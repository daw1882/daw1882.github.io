  1. GraphGrad

    Custom C++ tensor library with pybinding support that allows many standard tensor operations.

    • C++
    • Python
    • CUDA
    • OpenMP
    • Pybind

    GraphGrad

    Github Repo

    Description

    A custom-built C++ tensor library to perform matrix and tensor operations efficiently with automatic differentiation support for machine learning algorithms (a simplified version of PyTorch/TensorFlow). The library allows for multiple operations such as add, subtract, matmul, etc. and supports Python usage through pybindings. Multiple optimization techniques were implemented such as Lazy Evaluation, OpenMP CPU optimizations, SIMD computation, Common Subexpression Elimination, and GPU kernels. For more information and documentation for the library, check out the GitHub repository linked above.

    Why?

    This was a semester group project for a course on Machine Learning System Implementations and our group wanted to better understand what it took to implement the machine learning libraries we've so often used.

    My Contributions

    We collaborated to come up with the overall architecture design and I implemented many of the standard operations, some GPU kernels, OpenMP optimizations, a couple of the backpropogation functions, the majority of the pytests, and initial benchmark comparisons to evaluate our library's efficiency.

  2. Text Recovery from Palimpsests

    Spatial revising variational autoencoder for the purposes of recovering text of from Palimpsests, or old manuscripts which were chemically altered to be reused, using multi-spectral images.

    • Python
    • PyTorch
    • Computer Vision

    Text Recovery from Palimpsests

    Github Repo

    Description

    A PyTorch implementation of the Spatial Revising Variational Autoencoder from Yu et al. for extracting useful features from hyperspectral image data. After implementation, this model was then used to extract features from multi-spectral images of palimpsests in order to perform segmentation on images and recover hidden or damaged text.

    Why?

    This was a class project for a Neural Networks and Machine Learning course done in a group. For our project, we chose our topic to be text recovery from palimpsests as one of the group members was an image science major working on a Capstone project in the same topic and had access to multi-spectral image data for training and inference. While searching for potential model architectures that would work well, we came across a Spatial Revising Variational Autoencoder used for segmentation of hyperspectral images. Since this was exactly what we wanted to do, we began our project as an application study to apply this model to our topic.

    My Contributions

    I implemented the CNN module, parts of the LSTM module, modified the loss function, built most of the encoder and decoder, and trained and evaluated the model on our data.

  5. Traffic Camera Reprojection to Supplement Autonomous Perception

    A lightweight system for cooperative perception between LiDARs on autonomous vehicles and monocular traffic cameras.

    • Python
    • Computer Vision
    • CARLA
    • Research

    Traffic Camera Reprojection to Supplement Autonomous Perception

    Description

    A cooperative perception system attempting to combine data from monocular traffic cameras with LiDARs on autonomous vehicles in a Bird's Eye View format in order to improve autonomous vehicle perception and visibility. This uses the pinhole camera method to first estimate the objects positions for the camera view and then further refined by using a point alignment algorithm such as ICP to match the camera estimations with the vehicle's detections.

    Why?

    Traffic intersections are one of the most accident-prone areas on the road and contain many potential occlusions, or blind spots, for both human drivers and autonomous vehicles alike. One way to improve autonomous vehicle visibility is through the use of cooperative perception and combining what the vehicle sees with what another third party sensor can see. Since traffic cameras are already commonly found at many intersections, it would be widely beneficial to use these for cooperative perception.

    My Contributions

    This project started as a group class project with two others for an Autonomous Vehicles course and I expanded upon it for an independent study. I wrote the CARLA simulation code for collecting data and evaluating the system, implemented point alignment, and modified the pinhole camera code.

  8. Voice Cloning Resume Website

    Voice Cloning Resume Website

    Project Information

    Description

    A website demo for using ChatGPT to create a question-answer experience for asking questions about one's resume and receiving answers in a text-to-speech system using voice cloning to replicate the voice of the resume owner. Voice cloning was performed using the Tortoise-TTS model set up on HuggingFace Space.

    Why?

    As a group project for a speech processing course, we wanted to explore the topic of voice cloning and as we were nearing graduation, a practical application for a resume website was a fun use case to try it on. This allowed us to learn about the topic while also creating something a system to potentially use for ourselves in the future.

    My Contributions

    I managed the model interactions and designed the evaluation methodology for determining how well our voices were cloned. I additionally helped with report and presentation writing.
